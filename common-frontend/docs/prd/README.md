# PRD Docs

`docs/prd/` 存放使用方项目的需求、用户路径、业务规则和验收口径。模板只保留入口规则，不包含真实产品需求。

## 内容

- 原始需求、PRD、用户故事和用户路径。
- 业务规则、权限规则、状态流转和边界场景。
- 验收标准、设计输入、参考截图和需求变更记录。

## 不放

- 前端实现细节、组件 API、目录规范或编码规范。
- 接口契约、OpenAPI/Swagger、mock 数据或错误码清单。
- 当前执行状态、临时实现计划或 agent 过程记录。
- 未标明来源和状态的一次性讨论材料。

## 结构

```text
docs/prd/
  README.md
```

需求较多时，可按需增加 `V1.0/`、`feature-name/`、`changelog.md`。

## 维护规则

- 每轮成体系的需求优先放入版本目录，例如 `V1.0/`。
- 多文档专项需求优先收敛到专题目录，例如 `feature-name/`。
- 需求文档应说明来源、目标用户、核心路径、业务规则和验收口径。
- 已进入规格流程的内容，应与 OpenSpec 保持可追溯关系。
